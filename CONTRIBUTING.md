## ‚úçÔ∏è Submitting a Pull Request

### 1. Create a new branch

Use descriptive branch names such as:

- `feature/add-js-example`
- `fix/docs-typo`
- `improvement/php-snippet`

### 2. Make your changes

- Keep your commits small and focused.
- Follow the existing repository structure.

### 3. Push your branch

```bash
git push origin feature/your-feature-name
```
### 4. Open a Pull Request

When creating a PR:

- Use the PR template located in `.github/pull_request_template.md`
- Clearly explain **what you changed** and **why**
- Reference any related Issues or Discussions (e.g., `Closes #12`)
- Attach screenshots if your changes affect output, formatting, or examples
- Ensure your branch is up to date with `main`

---

- Approve and merge the PR when ready
